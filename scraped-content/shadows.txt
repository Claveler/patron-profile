Elevations and shadows help to provide visual feedback and focus through depth.

The UI components are typically flat, but depth levels can be added to establish visual relationships and hierarchy.

Our shadow system adheres to a consistent scale that enhances visual balance, thereby improving user comprehension and overall interface quality.

Specific intent

Shadows were initially created to enhance user comprehension and provide additional information on the interface. Each shadow elevation size serves a particular purpose and indicates possible interactions or user actions. Their placement within the interface is determined by their hierarchy of importance and their effect on user interaction and attention.

The use of shadows in the interface should follow certain rules:

Consistent use of shadows is important, especially when different elevations are present on the same row or plane.

Shadows should react to the component they are attached to.

Shadows are created by a combination of shape and light source, and it is crucial to maintain the same light source throughout the product. Using different light sources on the same interface can lead to user confusion.
